The command file, specified by option -c on the command line, is a text file consiting of commands that affect the mode of disassembly. These commands determine the label class that an assembly instruction, or a range of instructions of a given type will be disassembled. There are four different types of commands:

Comment Command Lines

A comment line can be inserted by beginning it with an asterisk (*). These lines will be ignored by OS9Disasm when processing.

Option Command Lines

An option line is a line with a plus (+) sign in column one, followed by a list of option parameters. These options are the sames as those you would use on the command line. Any will work here exept, of course, the c option. The reason for this is that this option would have to have been used to be reading this file in the first place, and only one command file is permitted.

It's quite useful to embed commands into this file in order to eliminate the need to include them on the shell command line each time OS9Disasm is executed.

Label Addressing Mode Specification Command Line

This defines the addressing mode by which this address or address range are specified. See Section 4.1, “Label Addressing Mode Specification Command Line” for a detailed explanation of this concept.

Data Area Boundary Specification Command Line

Definitions for areas that are data, and not code. Please refer to Section 4.2, “Data Area Boundary Specification Command Line” for more details on this.
