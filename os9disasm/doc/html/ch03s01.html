3.1. Invoking OS9Disasm

OS9Disasm is invoked by issuing the command in the following format: 


    os9disasm <options> [module-name] <options>

3.1.1. Options Used by OS9Disasm

OS9Disasm uses the following options. Some of these options will be more fully explained later. The options cannot be grouped, and must be preceded by a minus sign (-).

c[=]<command-file>Use <command-file> for disassembly commands. This is the only option that cannot be specified in the command file (which would be itself).
s[=]<label-file>Use <label-file> for label assignments
o[=]<asm-source-filename>Output assembly source code to filename (off by default)
lsGenerate short (6-char) labels (Default is 8 chars)
uFold output to upper-case
pw[=]<pagewidth>Set page width for listing output (Default=80)
pd[=]<pagedepth>Set page deptth for listing output (Default=80)
dDefine path to defs. This is the directory under which the default label files are found. The default is $HOME/coco/defs
3Disassemble for 6309 CPU. By default, OS9Disasm assumes 6809 code and ignores 6309 mnemonics, usually generating fcb's for these commands.
?Print a brief help screen.

Most of the above command options are self-explanatory. Briefly, you can (and ought to) use a single command file. This file provides all the disassembly details. It defines the class by which a label is defined at a given location, it defines data (non-code) regions in the program, and also can specify command-line options whereby you are not required to insert them on the command line each time you run OS9Disasm on that particular program.

You can use one or more label files (up to 16) where you can specify the name of labels generated during the disassembly. There are some label files used by default, where some standard labels are defined. The default label files used are dependent upon what OS the module was designed for.
