2.3. Building & Installing the Package

After you cd into the os9disasm directory, issuing the command make will build the program OS9Disasm. This will build the program, but won't install it.

To install the program, you can issue the command make install. This can be done either after the program has been built, or before. If it hasn't been built, issuing this command will cause it to be built before attempting to do the install. By default, this will install the OS9Disasm in the directory /usr/local/bin/. If you wish to install it elsewhere, then you can either simply copy the program OS9Disasm to whatever destination you want after running the command make, or you can edit the Makefile and change the line INSTALLDIR = /usr/local/bin to whatever you wish and then you can issue the command make install.

2.3.1. Installing the Standard Label Files

In the top level directory, there is a directory named defs. In this directory, you will find several files ending in "lbl". These are the standard label files. Copy all these files into $HOME/coco/defs/. This should work well for a single user system, or on any system where only one person is using this program.
